# REAGE – Rede de Apoio a Grupos de Estudo 🎓

O **REAGE** é uma plataforma concebida para conectar estudantes e otimizar a organização de grupos de estudo. Este projeto é o resultado de um ciclo completo de **Engenharia de Requisitos**, unindo técnicas de elicitação, análise de negócios e design de interface (UI/UX) para criar uma solução robusta e centrada no usuário.

## 🛠️ Processo de Engenharia de Software

O diferencial deste projeto foi a utilização de metodologias formais para garantir que o software resolvesse problemas reais:

1. **Brainstorming & Planejamento:** O grupo utilizou sessões de brainstorming para definir o escopo, público-alvo e os objetivos da aplicação.
2. **Elicitação de Requisitos:** Realização de **Entrevistas Técnicas** simuladas com stakeholders para validar necessidades e identificar dores dos usuários.
3. **Modelagem UML:** Criação de Diagramas de Caso de Uso para mapear as interações entre atores e o sistema.
4. **Refinamento Pós-Protótipo:** Ciclo de melhoria contínua onde os Requisitos Funcionais foram revisados e ajustados após a validação visual no Figma.

## 👥 Equipe e Contribuições

Este projeto foi desenvolvido de forma colaborativa, com responsabilidades bem definidas para simular um ambiente real de desenvolvimento:

### **Minhas Contribuições (Talita Lima):**
* **Regras de Negócio (RN):** Definição das diretrizes lógicas e restrições fundamentais do sistema.
* **Elicitação de Requisitos:** Atuação como entrevistadora na simulação com stakeholders, elaborando o roteiro e conduzindo a coleta de dados.
* **Refinamento de Requisitos:** Revisão e melhoria dos Requisitos Funcionais (RF) com base na análise do protótipo.
* **Arquitetura de Informação:** Auxílio na concepção estratégica do Design (UI/UX) para garantir a aderência às regras de negócio.

### **Colaboradores:**
* **Integrante 1:** Documentação teórica (Introdução, Descrição Geral e Coleta de Dados),Design e Desenvolvimento do protótipo de alta fidelidade no Figma.
* **Integrante 2:** Especificação detalhada de Requisitos Funcionais (RF).
* **Integrante 3:** Especificação de Requisitos Não Funcionais (RNF) e Validação do protótipo.
* **Integrante 4:** Elaboração do Diagrama de Casos de Uso e refinamento de RF.
